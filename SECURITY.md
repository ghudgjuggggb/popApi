# Security Policy

## Supported Versions

Мы поддерживаем следующие версии:

| Version | Supported |
|---------|-----------|
| 2.x | ✅ |
| 1.x | ❌ (устарело) |

## Отчёты об уязвимостях

**НЕ** открывайте публичный Issue для уязвимостей.

Вместо этого:

1. Email: ghudgjuggggb@github.com
2. Тема: "Security Issue"
3. Описание проблемы
4. Шаги для воспроизведения

Мы ответим в течение 48 часов.

## Что будет сделано

1. Мы подтвердим уязвимость
2. Создадим patch
3. Выпустим новую версию
4. Опубликуем security advisory
5. Отдадим credit авторам (если хотят)

## Практики безопасности

✅ Локальное хранение токенов
✅ Защита от traversal атак
✅ Валидация всех входов
✅ Регулярные обновления
✅ Аудит зависимостей

## Обновления

Регулярно обновляйте popApi:

```bash
npm update popapi
```
